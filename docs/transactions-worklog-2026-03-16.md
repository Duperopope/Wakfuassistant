# Wakfu Assistant - Transactions Worklog (2026-03-16)

## Scope
This update reworks the Transactions tab and the kama history pipeline for reliability, traceability, and UX.

## Main Changes
- Renamed and integrated Transactions tab in the main UI flow.
- Added interactive chart + history table for kama transactions.
- Added click on chart point -> auto-scroll to related history row.
- Added visible row highlight and pulse behavior with softer final styling.
- Added header-click sorting and persisted column layout behavior.
- Added support to remove a manual correction directly from the history table via context menu.

## Data Pipeline Rework
- Introduced normalized permanent journal:
  - `logs/permanent/kamas_events.jsonl`
  - generated by `core/permanent_journal.py`
- Each recorded event stores:
  - client time extracted from Wakfu logs
  - local machine timestamp
  - UTC timestamp
  - estimated atomic UTC timestamp and offset (NTP when available)
  - source file and source offset for traceability
- Added source-state tracking in `logs/permanent/recorder_state.json`.
- Ingestion now includes rotated log files (`.1`, `.2`, etc.) to extend history range.
- Added dedup fingerprints to reduce duplicate events across overlapping sources.

## Kamas Computation Logic
- Series reconstruction now uses correction anchors and current-kamas anchoring fallback.
- Avoided unstable reconstruction that could produce unrealistic visual drops.
- Kept non-negative display guard for chart output.

## Files Touched
- `core/permanent_journal.py` (new)
- `core/kamas_history.py`
- `ui/window.py`
- `ui/tabs/transactions.py`
- `ui/tabbar.py`

## Notes
- Runtime/local data files (logs, DB, caches) are intentionally excluded from this commit.
- If needed, next step can add a dedicated audit mode in Transactions to show source metadata per row.
