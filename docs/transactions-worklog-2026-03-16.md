# Wakfu Assistant - Transactions Worklog (2026-03-16)

## Scope
This update reworks the Transactions tab and the kama history pipeline for reliability, traceability, and UX.

## Main Changes
- Renamed and integrated Transactions tab in the main UI flow.
- Added interactive chart + history table for kama transactions.
- Added click on chart point -> auto-scroll to related history row.
- Added visible row highlight and pulse behavior with softer final styling.
- Added header-click sorting and persisted column layout behavior.
- Added support to remove a manual correction directly from the history table via context menu.

## Data Pipeline Rework
- Introduced normalized permanent journal:
  - `logs/permanent/kamas_events.jsonl`
  - generated by `core/permanent_journal.py`
- Each recorded event stores:
  - client time extracted from Wakfu logs
  - local machine timestamp
  - UTC timestamp
  - estimated atomic UTC timestamp and offset (NTP when available)
  - source file and source offset for traceability
- Added source-state tracking in `logs/permanent/recorder_state.json`.
- Ingestion now includes rotated log files (`.1`, `.2`, etc.) to extend history range.
- Added dedup fingerprints to reduce duplicate events across overlapping sources.

## Kamas Computation Logic
- Series reconstruction now uses correction anchors and current-kamas anchoring fallback.
- Avoided unstable reconstruction that could produce unrealistic visual drops.
- Kept non-negative display guard for chart output.

## Files Touched
- `core/permanent_journal.py` (new)
- `core/kamas_history.py`
- `ui/window.py`
- `ui/tabs/transactions.py`
- `ui/tabbar.py`

## Notes
- Runtime/local data files (logs, DB, caches) are intentionally excluded from this commit.
- If needed, next step can add a dedicated audit mode in Transactions to show source metadata per row.

## Follow-up Refinements (same session)
- Time window controls were made data-aware:
  - only proposes ranges that are truly available from current history depth and recency
  - keeps smart wrap behavior between short ranges and `all`
- Period filtering now uses real-time window logic (`now - range`) and includes visual boundary anchors:
  - left anchor at cutoff
  - right anchor at now
  - avoids fake transaction rows while keeping the visible window aligned with real time
- Chart rendering was changed from linear interpolation to staircase mode to better represent discrete transaction events.
- Wheel zoom behavior was expanded:
  - deeper zoom-in enabled (lower minimum visible points)
  - zone-aware zoom interactions:
    - right side anchors to present
    - center stays cursor-centered
    - left side keeps past-oriented context
  - dynamic range expansion on zoom-out beyond selected period with progressive minute step growth
- Range label now reflects the actual visible span in viewport instead of only the selected mode label.
- Tooltip improvements:
  - renamed from `Solde estime` to `Solde`
  - added real per-step kama delta (`Difference: +/-N`) computed from previous point
- Vertical scaling was adjusted to keep step heights visually relative to absolute kama magnitude and reduce exaggerated micro-variation jumps.

## Final UI Pass (same session)
- Metrics panel was reorganized into:
  - one combined card for Gains + Losses
  - one combined card for Net + Current Kamas
  - two intentionally empty placeholder cards for future features
- Kama symbol display was standardized across Transactions:
  - chart tooltip (`Solde`, `Difference`)
  - history amounts and history column title
  - correction deletion confirmation text
